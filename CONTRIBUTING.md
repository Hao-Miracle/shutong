# 贡献指南

感谢你有兴趣参与书童项目！

## 如何贡献

### 提交问题和建议

如果你在使用中发现bug或有改进建议，请提交 Issue。

### 提交内容

可以贡献的方向：
- 修正文档中的错别字或表达不清
- 补充FAQ中的常见问题
- 改进Prompts的措辞
- 增加更多的书型策略
- 补充虚构示例

### 不允许的贡献

- 包含真实个人数据的示例（请使用虚构内容）
- 商业用途的修改（见许可证）

## Pull Request 流程

1. Fork 本仓库
2. 创建你的特性分支
3. 提交改动
4. 确保没有包含个人数据
5. 创建 Pull Request

## 格式要求

- 中文文档使用简体中文
- 英文文档与中文文档同步更新
- Markdown 格式，不超 80 列宽
