# 工作流：读书 → 复盘 → 转化

每本书跑完这三步。每一步都有明确的"谁做什么"。

---

## 第一步：读书

> 目标：把书**读懂**，产出**你自己的**读书笔记。

### 流程

```
你给书名 → AI先读完（预习） → AI诊断书型 → AI给阅读方案
→ 你自己读 + 记笔记 → 你把笔记丢给AI → AI用问题引导你
→ 循环直到读完
```

### 子动作

| 子动作 | 谁来做 | 产出 |
|--------|--------|------|
| 诊断书类型 | AI | 书型标签 |
| 定制阅读策略 | AI | 阅读方案 |
| 完整读完全书（预习） | AI | 全书理解（内部） |
| 读书 + 写笔记 | **你** | `笔记.md` |
| 提交进度/笔记 | **你** | 交互输入 |
| 基于笔记引导讨论 | AI | 理解深度 |

### 笔记要求

**笔记是你写的，AI不会替你写一个字。**

合格笔记的三个标准：
1. **有自己的话** —— 不是抄原文，是你理解后的表达
2. **有个人关联** —— 联系自己的经历、工作、见过的事
3. **有思考痕迹** —— 有疑问、有反驳、有"这不合理"

> 合格笔记 = AI看完后能追问你至少3个问题。

---

## 第二步：复盘

> 目标：从笔记中提炼概念和模型，把书里的东西变成可带走的思维工具。

### 四层跃迁

```
你的原始笔记（第一步产出）
    ↓ 过滤
关键信息
    ↓ 抽象（你命名定义 + AI追问精确度）
核心概念（你自己的话）
    ↓ 结构化（你构建 + AI挑战漏洞）
思维模型（你可复用的工具）
    ↓ 验证（你费曼输出 + AI扮不懂的朋友）
可迁移智慧
```

### 什么是"概念"

概念是一个脱离具体书籍存在的、可被多本书引用的知识节点。

**是概念的：**
- 互惠原理
- 沉没成本谬误
- 第一性原理

**不是概念的：**
- 书里的某个故事（那是案例）
- 作者的个人经历（那是背景）
- 你的读后感（那是反思）

### 概念存储

概念提炼出来后，存到 `概念图谱/概念/` 中。每个概念一个文件。
多本书可以引用同一个概念，形成跨书的知识网络。

---

## 第三步：转化

> 目标：从"知道"到"做到"。

### 流程

```
定义行动 → 执行 → 验证 → 封印
```

### 什么是好的行动

| 标准 | ✅ 合格 | ❌ 不合格 |
|------|--------|---------|
| 具体 | "下周开会时尝试先问对方需求，再提方案" | "以后要多倾听" |
| 可验证 | "3天后检查是否做到了" | "慢慢改进" |
| 关联书本 | 行动明确来自某本书的某个概念 | 跟书没有关系 |

### 封印

当你的行动验证通过后，将概念和模型正式"封印"到思维殿堂中。封印意味着：
- 这个概念已经过你的实践验证
- 你可以在日常中随时调用它
- 它成为你思维殿堂的一块砖

---

## 完成后

一本书完成 读书→复盘→转化 三步后：

1. 书中的概念已进入你的概念库
2. 你至少完成了一项行动验证
3. 选下一本书，重复流程
4. 累积的概念和模型互相连接，形成你的思维格栅

**这就是量变到质变的过程。**
