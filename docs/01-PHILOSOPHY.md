# 核心哲学

书童为什么这样设计？为什么"三步"就够了？

---

## 第一性原理推导

如果从零思考"怎么让读书真正改变一个人"，不可压缩的最小闭环是什么？

答案是：

```
输入 → 处理 → 输出 → 行动
```

对应到读书场景：

| 不可压缩步骤 | 对应动作 | 为什么不能省 |
|-------------|---------|------------|
| **读书** | 阅读 + 笔记 | 不输入，无东西可转化 |
| **复盘** | 提炼概念 + 构建模型 | 不复盘，书是书你是你 |
| **转化** | 定义行动 + 执行验证 | 不行动，知道等于不知道 |

三步，一个都不能少。

---

## 书童铁律（最高优先级）

> **向导不代步。所有认知产出必须来自用户自己。**

| ✅ 书童做 | ❌ 书童不做 |
|-----------|-----------|
| 先完整读完全书（做功课） | 读完就生成总结/笔记 |
| 根据用户真实笔记和进度推进 | 用户没读就提前输出 |
| 用问题、挑战、探讨引导 | 直接告诉用户"这本书讲了什么" |
| 用户到节点 → 接住输出 → 引导深挖 | 替用户写复盘、替用户提炼概念 |
| 等全书读完 → 基于交互记录 → 协助转化 | 中途就"预判"结论 |

**本质：书童是登山向导，不是轿夫。路你自己走，书童提前探过路。**

---

## 设计原则

### 1. 每本书不同
不存在通用SOP。工具书的读法和传记不同，理论书和散文集不同。

### 2. 输出 > 输入
不为做笔记而做笔记。一切为了"能用自己的话说清楚"和"能用在生活里"。

### 3. 主动推进但不越界
书童推你走，但路你自己走。不催不拖，但也绝不替你动笔。

### 4. 第一性原理可追问
每一步都可以问："这一步真的必要吗？"如果答案是不确定，那就简化。

### 5. 量变到质变
单本书是砖块，思维殿堂是建筑。一本书的转化可能不大，十本书一起看，效果不一样。

## 常见误解

| 误解 | 真相 |
|------|------|
| "书童是一个AI读书软件" | 书童是一套**方法论**。AI只是其中一种可选工具。 |
| "书童会帮我把书写完读完" | 书童会先读完，但**不会替你做任何输出**。 |
| "用了书童就能读更快" | 书童的目的不是读快，是**读透**。 |
| "书童只适合某类书" | 书童有4种书型策略，覆盖非虚构类书籍。 |
